# Story 4.5 - Runtime Constant Consolidation

**Status:** Not Started

## Overview

Extract shared runtime constants (clipboard capabilities, loader timeouts,
status labels, error codes) into a single module to reduce duplication and
ensure consistency across background, popup, and options surfaces.

## Objectives

- Identify scattered constants across codebase
- Create centralized constants module
- Update all consumers to use shared constants
- Add TypeScript types for constant values

## Acceptance Criteria

- [ ] Single `src/lib/constants.ts` (or similar) exports all shared values
- [ ] Clipboard capability constants centralized
- [ ] Timeout values (loader, debounce) in one place
- [ ] Status labels and error codes unified
- [ ] No duplicate magic strings/numbers in codebase
- [ ] TypeScript enums or const objects for type safety

## Risks & Mitigations

- **Import cycles:** Structure module to avoid circular dependencies
- **Bundle size:** Tree-shaking should eliminate unused constants
